How to Access PDF Studio

Pick the method that works best for you — Docker is the easiest, local Python gives you full control, and Heroku lets you share it with anyone.

1

Install Docker Desktop

Download and install Docker Desktop for your operating system (Windows, macOS, or Linux). Start it and wait until it shows "Docker is running".

2

Clone the repository

Open your terminal (or Command Prompt on Windows) and run:

# Download the project git clone https://github.com/taiyeba-dg/lakshya-pdf.git cd lakshya-pdf

Don't have git? Download Git, or download the project as a ZIP from GitHub (click Code → Download ZIP) and unzip it.

3

Start the app

# Build and start (first run takes ~2 minutes) docker-compose up --build

Wait until you see Application startup complete in the terminal.

4

Open in your browser

Your app is running at http://localhost:8000

To stop: press Ctrl+C in the terminal, then run docker-compose down.

1

Install prerequisites

You need Python 3.10+ and two system libraries. Install them for your OS:

# Ubuntu / Debian / WSL (Windows Subsystem for Linux) sudo apt-get update && sudo apt-get install -y \ python3 python3-pip poppler-utils libreoffice # macOS (Homebrew) brew install python poppler brew install --cask libreoffice

Windows users: Install WSL (Windows Subsystem for Linux) first, then use the Ubuntu commands above.

2

Clone and install Python packages

git clone https://github.com/taiyeba-dg/lakshya-pdf.git cd lakshya-pdf # (Recommended) Create a virtual environment python3 -m venv venv source venv/bin/activate # Windows: venv\Scripts\activate # Install Python dependencies pip install -r backend/requirements.txt
3

Start the server

uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000
4

Open in your browser

Your app is running at http://localhost:8000

Important: Always open the http://localhost:8000 URL — do not open the HTML files directly from the file system, or the tool pages will fail to call the API.

1

Create a free Heroku account & install the CLI

Sign up at heroku.com, then install the Heroku CLI.

2

Clone, create app, and deploy

git clone https://github.com/taiyeba-dg/lakshya-pdf.git cd lakshya-pdf heroku login heroku create your-app-name # pick a unique name git push heroku main heroku ps:scale web=1
3

Open the live URL

heroku open # opens https://your-app-name.herokuapp.com
Your app is live at https://your-app-name.herokuapp.com — share this link with anyone!

Heroku's ephemeral filesystem auto-deletes uploaded files between restarts. This is actually a privacy feature!

🔧 Troubleshooting

Port 8000 is already in use
Use a different port: add --port 8080 to the uvicorn command, then open http://localhost:8080.

With Docker, edit docker-compose.yml and change the left side of the ports mapping:
ports: - "8080:8000" # host:container
"No module named 'pikepdf'" or import error
Install dependencies in the right Python environment:
pip install -r backend/requirements.txt
If you use a virtual environment, make sure it's activated (source venv/bin/activate) before installing and before starting the server.
PDF to Image returns an error
poppler-utils must be installed at the system level (not via pip):
# Ubuntu/Debian/WSL sudo apt-get install poppler-utils # macOS brew install poppler
Restart the server after installing.
PDF to Word doesn't work
libreoffice must be installed:
# Ubuntu/Debian/WSL sudo apt-get install libreoffice # macOS brew install --cask libreoffice
Restart the server after installing.
Page loads but tool pages give API errors
Always open the app through the server URL (http://localhost:8000), not by opening the HTML files directly from your file system (file://…). The tool pages make API calls to the backend and only work when served through the server.
Docker build fails or takes very long
Make sure Docker Desktop is running. Then try a clean rebuild:
docker-compose down docker-compose up --build --force-recreate
The first build installs LibreOffice which is large (~200 MB) — subsequent starts are much faster.
